Create `scripts/epistemic_contract.py` with the following functions:

**1. `find_cross_contract_entities(nodes: list[dict]) -> list[dict]` (XREF-01)**
- Iterate nodes; those with `source_documents` list containing 2+ entries are cross-contract
- Also check links: build a `entity_id -> set(source_documents)` map from link `source_document` fields, since sift-kg may track provenance on links rather than nodes
- Return list of dicts: `{"entity_id", "entity_name", "entity_type", "contracts": [...], "contract_count": N}`
- Sort by contract_count descending

**2. `load_conflict_rules(domain_yaml_path: Path) -> dict` (supports XREF-02)**
- Load domain.yaml with `yaml.safe_load()`
- Return `data.get("conflict_rules", {})`
- If no conflict_rules key, return empty dict (graceful for domains without rules)

**3. `detect_conflicts(nodes: list[dict], links: list[dict], rules: dict) -> list[dict]` (XREF-02)**
- For each rule in rules, find candidate node pairs:
  - **exclusive_use**: Find CLAUSE nodes with `attributes.clause_type == "exclusivity"`, find VENUE nodes they restrict (via RESTRICTS links). If two different contracts both restrict the same venue space, flag conflict.
  - **schedule_contradiction**: Find DEADLINE nodes, group by `attributes.what_is_due` (fuzzy: lowercase strip). Within each group, compare `attributes.date` values across contracts. If different dates for same deliverable from different contracts, flag.
  - **term_contradiction**: Find OBLIGATION and CLAUSE nodes, group by similar `attributes.action` or `attributes.key_terms`. If conflicting terms found (one restricts what another permits), flag.
  - **cost_budget_mismatch**: Find COST nodes, compare amounts for similar services across contracts. Also compare against reference nodes (from Master doc) if `compare_against == "reference"`.
- Each finding dict: `{"id": "conflict:{rule_name}:{seq}", "type": rule_name, "severity": rule.severity, "description": str, "entities_involved": [id1, id2], "contracts_involved": [doc1, doc2], "evidence": {"source_a": {...}, "source_b": {...}}, "suggested_action": str (from template), "confidence": float}`
- Per D-06: rules are transparent and auditable.
- Handle missing attributes gracefully: if a node lacks the required attribute for comparison, skip that pair and log as INFO finding ("Insufficient data for conflict check on entity X"). Per research Pitfall 2, check relation context not just entity names.

**4. `import_master_doc(master_path: Path, graph_data: dict) -> tuple[list[dict], list[dict]]` (XREF-03)**
- Parse `Sample_Conference_Master.md` (or any reference doc) into PLANNING_ITEM reference nodes
- Extract sections: look for markdown headers (`##`, `###`), budget items (lines with `$` amounts), timeline items (lines with dates), and requirement items (lines with action verbs)
- Each reference node: `{"id": "ref:{slugified_description}", "name": description, "entity_type": "PLANNING_ITEM", "source": "reference", "source_document": "Sample_Conference_Master", "attributes": {parsed attrs}, "confidence": 0.5}`
- Per D-08: visually distinct, source="reference", non-authoritative
- Per D-09: contracts override; findings framed as "Master doc expects X, no contract covers it"
- Return (reference_nodes, reference_links) where links connect to matching contract entities

**5. `find_coverage_gaps(ref_nodes: list[dict], contract_nodes: list[dict], links: list[dict]) -> list[dict]` (XREF-03)**
- For each PLANNING_ITEM ref_node, search contract_nodes for matching coverage:
  - Budget items: find COST nodes covering similar services
  - Timeline items: find DEADLINE nodes for similar milestones
  - Requirement items: find OBLIGATION nodes for similar actions
- Matching is fuzzy: lowercase keyword overlap, with a threshold (e.g., 2+ shared significant words after stopword removal)
- Gaps = ref_nodes with no matching contract entity
- Each gap: `{"id": "gap:{seq}", "type": "coverage_gap", "severity": "WARNING" (or "CRITICAL" if budget/safety), "description": "Master doc expects '{ref.name}', no contract covers it", "reference_entity": ref.id, "expected_coverage": ref.attributes, "suggested_action": "Add contract coverage for: {ref.name}"}`

**6. `score_risks(conflicts: list[dict], gaps: list[dict]) -> list[dict]` (XREF-04)**
- Aggregate all findings into risk items per D-10, D-12
- CRITICAL: exclusive_use conflicts, term contradictions, gaps in safety/insurance coverage
- WARNING: schedule contradictions, cost mismatches, budget gaps
- INFO: minor gaps, informational notes, insufficient data notes
- Each risk: `{"id": "risk:{seq}", "severity": str, "source_type": "conflict"|"gap"|"cross_check", "source_id": str, "description": str, "suggested_action": str, "contracts_affected": [...]}`
- Per D-11: each finding includes a brief suggested action

**7. `analyze_contract_epistemic(output_dir: Path, graph_data: dict, master_doc_path: Path | None = None) -> dict`**
- Main entry point called by label_epistemic.py dispatcher
- Load conflict_rules from domain.yaml via `resolve_domain("contract")` -> load YAML -> get conflict_rules
- Call `find_cross_contract_entities(nodes)`
- Call `detect_conflicts(nodes, links, rules)`
- If master_doc_path provided and exists: call `import_master_doc()` then `find_coverage_gaps()`
- If master_doc_path not provided: skip gap analysis, log INFO
- Call `score_risks(conflicts, gaps)`
- Set `epistemic_status` on links: "asserted" for high-confidence contract terms, "contested" for terms involved in conflicts, "unverified" for low-confidence
- Build and return claims_layer dict with `super_domain.domain = "contract"` per research Pitfall 4
- Per D-01: all findings in claims_layer.json as Super Domain overlays

Follow project conventions: snake_case, pathlib.Path, type hints on all signatures, docstrings, `json.dumps(indent=2)`, error dicts not exceptions.
