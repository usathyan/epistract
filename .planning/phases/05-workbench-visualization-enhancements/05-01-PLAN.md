<objective>
Improve visual legibility of the workbench graph panel without adding new UI chrome. Three sub-enhancements that all live in `buildGraph()` in `graph.js`:

1. Label halo + scaling so node labels remain readable on dense graphs and auto-hide at far zoom (UI-SPEC Typography, RESEARCH Pattern 2).
2. Degree-based node sizing so high-connectivity hub nodes are visually prominent (RESEARCH Pattern 4, UI-SPEC Interaction Contracts "Degree-Based Node Sizing").

Purpose: These three changes share the same file and function (`buildGraph()`) and are pure configuration — no event handlers, no new DOM, no state. Isolating them in Wave 1 lets Plan 02 focus entirely on interactive pinning + toolbar on top of a stable baseline.

Output: Updated `examples/workbench/static/graph.js` with upgraded node font config, per-node degree-scaled `size`, and explicit `interaction` options block. No changes to `index.html`, `style.css`, or any Python file.

    ```javascript
    // Pre-compute node degree (total incoming + outgoing edges) for visual sizing.
    // Range: 8px (isolated node) to 24px (highest-degree hub). See UI-SPEC Degree-Based Node Sizing.
    const degreeMap = {};
    allEdges.forEach(e => {
        degreeMap[e.source] = (degreeMap[e.source] || 0) + 1;
        degreeMap[e.target] = (degreeMap[e.target] || 0) + 1;
    });
    const maxDegree = Math.max(...Object.values(degreeMap), 1);
    ```

    Then replace the hardcoded `size: 12,` in the node mapping with:

    ```javascript
    size: 8 + Math.round(((degreeMap[n.id] || 0) / maxDegree) * 16),
    ```

    This yields the range 8px (no edges) to 24px (highest degree), matching the UI-SPEC formula exactly. A graph with no edges at all still renders at 8px because `maxDegree` is clamped to a minimum of 1.
