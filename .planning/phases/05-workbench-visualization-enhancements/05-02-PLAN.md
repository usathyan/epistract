## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User browser (DOM) → vis-network DataSet | Node IDs in `pinnedNodes` Set come from `params.nodes` which vis-network populates from internal DataSet IDs, not user-typed input. |
| DOM event listeners (window.resize, click) | Global listeners added once per `buildGraph()` call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | `dragEnd` handler writing to `visNodes.update()` | mitigate | Always use immutable DataSet `update()` — never `visNodes._data[id].fixed = ...`. Acceptance criterion enforces `grep -c 'visNodes._data\['` returns 0. |
| T-05-05 | Denial of Service | `window.addEventListener('resize', ...)` accumulating across `buildGraph()` re-invocations | accept | `buildGraph()` is called at most once per page load in the current codebase (only from `loadGraphData()` success path). Adding `{ once: true }` would break the intended resize behavior. Acceptable risk; revisit if multi-graph-per-session lands. |
| T-05-06 | Information Disclosure | Node popover leaking attribute values | mitigate | Existing `safeName` HTML-escape in `showNodePopover()` (graph.js lines 240–241) MUST remain intact. This plan's resize handler does not touch popover content — it only calls the existing `hideNodePopover()`. Verified by acceptance criterion that keeps existing click/doubleClick handlers untouched. |
| T-05-07 | Spoofing | Hardcoded accent hex `#4a6cf7` vs. CSS variable | accept | Per UI-SPEC Color section line 95, the accent value is canonical. Using a string literal instead of `getComputedStyle` makes pinning 10× faster and removes a cross-browser edge case. Aesthetic drift risk if the CSS token ever changes — addressed by keeping the literal consistent with `--color-accent` in style.css `:root`. |
</threat_model>
