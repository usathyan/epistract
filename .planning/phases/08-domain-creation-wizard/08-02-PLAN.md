    Create `core/domain_wizard.py` with the following functions. Per D-01 through D-16:

    **Module structure:**
    ```python
    #!/usr/bin/env python3
    """Domain creation wizard for epistract knowledge graph framework.

    Analyzes sample documents via multi-pass LLM analysis to propose a domain
    schema, then generates a complete domain package (domain.yaml, SKILL.md,
    epistemic.py, references/) ready for the standard pipeline.

    Usage:
        from core.domain_wizard import analyze_documents, generate_domain_package

        # Step 1: Analyze documents and get schema proposal
        proposal = analyze_documents(doc_paths, domain_description)

        # Step 2: Generate the full domain package
        result = generate_domain_package(domain_name, proposal, output_dir)
    """
    ```

    **Constants and imports:**
    - `from __future__ import annotations`
    - `import ast`, `import importlib.util`, `import json`, `import re`, `import yaml`
    - `from pathlib import Path`
    - `from core.domain_resolver import DOMAINS_DIR, list_domains, DOMAIN_ALIASES`
    - `MIN_SAMPLE_DOCS = 2` (per D-02)
    - `MAX_ENTITY_TYPES = 15`, `MAX_RELATION_TYPES = 20` (calibration bounds from research)

    **Function 1: `read_sample_documents(doc_paths: list[Path]) -> list[dict]`**
    - For each path, read the file text (plain `Path.read_text()` for .txt, attempt sift_kg reader for others)
    - Return list of `{"path": str, "text": str, "char_count": int}`
    - Raise `ValueError` if fewer than `MIN_SAMPLE_DOCS` readable documents (per D-02). Include warning message suggesting 3-5 docs

    **Function 2: `build_schema_discovery_prompt(doc_text: str, domain_description: str) -> str`**
    - Returns the Pass 1 prompt for LLM schema discovery from a single document
    - Prompt asks LLM to identify candidate entity types and relation types
    - Include the domain_description as context (per D-03)
    - Request JSON output format: `{"entity_types": [{"name": "TYPE_NAME", "description": "...", "examples": ["..."]}], "relation_types": [{"name": "REL_NAME", "description": "...", "source_type": "...", "target_type": "..."}]}`
    - Include instruction: "Propose 5-15 entity types and 5-20 relation types. Merge similar concepts into one type."

    **Function 3: `build_consolidation_prompt(candidates: list[dict], domain_description: str) -> str`**
    - Returns the Pass 2 prompt for cross-document consolidation
    - Takes the merged candidates from all Pass 1 results
    - Asks LLM to deduplicate synonyms, pick canonical SCREAMING_SNAKE_CASE names, write descriptions
    - Request same JSON output format as Pass 1 but consolidated

    **Function 4: `build_final_schema_prompt(consolidated: dict, domain_description: str) -> str`**
    - Returns the Pass 3 prompt for final schema proposal
    - Validates each relation references valid entity types
    - Request output matching domain.yaml format:
      ```json
      {
        "entity_types": {"TYPE_NAME": {"description": "..."}},
        "relation_types": {"REL_NAME": {"description": "..."}}
      }
      ```

    **Function 5: `check_domain_exists(domain_name: str) -> bool`**
    - Check if domain_name exists in `list_domains()` or `DOMAIN_ALIASES` (per D-16)
    - Return True if collision detected

    **Function 6: `generate_domain_yaml(domain_name: str, description: str, system_context: str, entity_types: dict, relation_types: dict) -> str`**
    - Generate domain.yaml content as YAML string (per D-05, contracts-level quality)
    - Use `yaml.safe_dump()` with `default_flow_style=False, sort_keys=False, allow_unicode=True`
    - Schema structure: `name`, `version` ("1.0.0"), `description`, `system_context`, `entity_types`, `relation_types`

    **Function 7: `generate_skill_md(domain_name: str, system_context: str, entity_types: dict, relation_types: dict, extraction_guidelines: str) -> str`**
    - Generate SKILL.md content as markdown string (per D-06, contracts SKILL.md complexity)
    - Format: heading, system context paragraph, Entity Types table, Relation Types table, Extraction Guidelines section

    **Function 8: `generate_epistemic_py(domain_slug: str, entity_types: dict, contradiction_pairs: list[tuple[str, str]], gap_target_types: dict, confidence_thresholds: dict) -> str`**
    - Generate epistemic.py as Python source code string (per D-09, D-10, D-11)
    - The `domain_slug` is the domain name with hyphens replaced by underscores
    - Function name: `analyze_{domain_slug}_epistemic(output_dir: Path, graph_data: dict) -> dict`
    - Include all four patterns: contradiction detection, confidence calibration, gap analysis, cross-document linking
    - Use the `contradiction_pairs`, `gap_target_types`, and `confidence_thresholds` as domain-specific injections (per D-10)
    - Return dict must have keys: `metadata`, `summary`, `base_domain`, `super_domain`
    - Template should be ~100-150 lines of working Python

    **Function 9: `generate_reference_docs(entity_types: dict, relation_types: dict) -> tuple[str, str]`**
    - Returns `(entity_types_md, relation_types_md)` (per D-07)
    - Format matches `domains/contracts/references/entity-types.md`: `## TYPE_NAME\nDescription...\n`

    **Function 10: `validate_generated_epistemic(code: str, domain_slug: str) -> dict`**
    - Validate generated epistemic.py (per D-12):
      1. `ast.parse(code)` -- syntax check
      2. Write to temp file, `importlib.util.spec_from_file_location` -- import check
      3. Check `analyze_{domain_slug}_epistemic` function exists
      4. Dry-run with `{"nodes": [], "links": []}` -- returns valid dict with required keys
    - Return `{"valid": True}` or `{"valid": False, "error": "..."}`
    - Clean up temp file after validation

    **Function 11: `write_domain_package(domain_name: str, domain_yaml: str, skill_md: str, epistemic_py: str, entity_types_md: str, relation_types_md: str) -> Path`**
    - Write all files to `DOMAINS_DIR / domain_name /` (per D-15)
    - Create `references/` subdirectory
    - Create `__init__.py` (empty)
    - Return the domain directory path

    **Function 12: `build_epistemic_prompt(entity_types: dict, relation_types: dict, domain_description: str, sample_excerpts: list[str]) -> str`**
    - Returns prompt for LLM to generate domain-specific epistemic parameters
    - Asks for: contradiction_pairs, gap_target_types, confidence_thresholds
    - Include sample document excerpts so LLM understands the domain's contradiction patterns (avoid Pitfall 5)
    - Request JSON output format

    All functions use `pathlib.Path` throughout (no `os.path`). All have docstrings with Args/Returns. Type hints with `|` union syntax.
