# Phase 9 — UI Design Contract

> Visual and interaction contract for workbench generalization and domain template system. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS) |
| Icon library | Unicode emoji (existing: sidebar nav icons) |
| Font | `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` (body), `"SF Mono", "Fira Code", "Consolas", monospace` (data) |

**Note:** This phase does not introduce a new design system. It generalizes the existing V1 workbench CSS custom property system to be domain-driven via `template.yaml`. All values below document the existing system that must be preserved and made configurable.

| Token | Value | Usage |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background, chat panel, card backgrounds |
| Secondary (30%) | `#1a1a2e` | Sidebar background, chat input bar, dashboard table headers |
| Accent (10%) | `#4a6cf7` | Send button, active tab underline, citation links, focus rings, "Ask about" button |
| Destructive | `#dc3545` | Critical severity badges, critical decision cards |

Accent reserved for: Send button, active panel tab border, graph search focus ring, citation links, "Ask about node" button, starter card hover border. Never used for body text or backgrounds.

### Severity Palette (existing, unchanged)

| Severity | Background | Text | Border |
|----------|------------|------|--------|
| Critical | `#fef2f2` | `#dc3545` | `#dc3545` |
| Warning | `#fffbeb` | `#d97706` | `#d97706` |

### Entity Color Palette (domain-driven)

The following 12-color palette is the default auto-assignment palette for domains that do not declare explicit `entity_colors` in `template.yaml`. Colors are assigned by modular index over entity types sorted alphabetically.

```
#6366f1, #f59e0b, #ef4444, #10b981, #8b5cf6,
#06b6d4, #64748b, #ec4899, #f97316, #14b8a6,
#a855f7, #0ea5e9
```

When a domain provides `entity_colors` in `template.yaml`, those override the palette for the specified types. Unlisted types fall back to the palette by index.

## Interaction Contracts

### Template Loading Flow

1. Server starts with `--domain {name}` argument (or no argument for generic)
2. Server loads `domains/{name}/workbench/template.yaml` if it exists
3. Falls back to `GENERIC_TEMPLATE` defaults for missing fields
4. Template data available via `GET /api/template` endpoint
5. Frontend fetches `/api/template` on init, populates all dynamic copy

### Dynamic Entity Toggles

1. Frontend fetches entity types from `/api/graph/entity-types` (existing endpoint)
2. Renders toggle buttons dynamically (replacing hardcoded HTML buttons)
3. Each button gets color from template `entity_colors` or default palette by index
4. Sidebar legend also rendered dynamically from same data

### Dashboard Panel

| Condition | Behavior |
|-----------|----------|
| Domain provides `workbench/dashboard.html` | Load and inject as dashboard content |
| Domain provides no dashboard | Auto-generate summary: entity counts by type table, top 10 most-connected nodes, relationship count |
| Generic fallback | Same auto-generated summary |

### Telegram Bot Interactions

| Command | Response |
|---------|----------|
| `/start` | Welcome message with domain title + up to 5 starter questions as numbered list |
| `/help` | "Send any question about the knowledge graph. I'll answer using {title} data." |
| Free text | LLM response using domain persona, graph data context, streaming disabled (full message) |

## CSS Custom Property Contract

The existing CSS custom property system in `style.css` is preserved. The following properties become domain-configurable via the template API:

| Property | Default | Overridable via Template |
|----------|---------|------------------------|
| `--color-dominant` | `#ffffff` | No (structural) |
| `--color-secondary` | `#1a1a2e` | No (structural) |
| `--color-accent` | `#4a6cf7` | No (structural) |
| `--color-destructive` | `#dc3545` | No (structural) |
| Entity type CSS vars (`--entity-*`) | Removed from CSS | Yes, via JS from template `entity_colors` |
| All spacing tokens | Unchanged | No (structural) |
| All typography tokens | Unchanged | No (structural) |

Entity colors move from CSS custom properties to JavaScript runtime assignment from the template API response. The `--entity-*` CSS variables in `style.css` are removed; colors are applied inline via JS (matching existing `graph.js` pattern).
