\subsection{Domain Pluggability}

Epistract's architecture separates the extraction pipeline from domain-specific knowledge through a three-layer design:

\begin{enumerate}
  \item \textbf{Core Pipeline} (\texttt{core/}): Domain-agnostic document ingestion, entity extraction orchestration, graph construction (via sift-kg), and epistemic analysis dispatch.
  \item \textbf{Domain Configurations} (\texttt{domains/}): Pluggable packages containing entity/relation schemas (YAML), extraction prompts (SKILL.md), and domain-specific epistemic rules (Python).
  \item \textbf{Example Consumers} (\texttt{examples/}): Application frontends (web dashboard, Telegram bot) that work with any domain's graph output.
\end{enumerate}

Adding a new domain requires creating a configuration package --- no modifications to the core pipeline. A domain creation wizard (\texttt{/epistract:domain}) automates this process by analyzing sample documents and generating a complete domain package.
```

\subsection{Case Study 2: Event Contract Management}

To validate Epistract's domain-agnostic capabilities, we applied the framework to a corpus of 62 event planning contracts for a multi-day convention (PDFs, spreadsheets, and email correspondence).

\textbf{Domain Configuration.} We defined a contract domain schema with 11 entity types (Party, Obligation, Deadline, Cost, Clause, Service, Venue, Committee, Person, Event, Stage) and 11 relation types (OBLIGATES, CONFLICTS\_WITH, DEPENDS\_ON, COSTS, PROVIDES, RESTRICTS, ASSIGNS, CHAIRS, SCHEDULES, LOCATED\_AT, STAGES).

\textbf{Extraction Results.} The pipeline extracted 341 nodes and 663 edges from the 62-document corpus. Entity resolution merged variant references (e.g., different name spellings for the same vendor) into canonical nodes.

\textbf{Epistemic Analysis.} Domain-specific epistemic rules detected 53 cross-contract conflicts --- including overlapping exclusive-use claims for venue spaces, incompatible scheduling requirements, and contradictory terms between related contracts. Gap analysis identified event requirements lacking corresponding contract coverage.

\textbf{Framework Validation.} The contract domain was created using the same pipeline infrastructure built for drug discovery, requiring only a new domain configuration package. No core pipeline modifications were necessary, confirming the framework's domain-agnostic design.
```
