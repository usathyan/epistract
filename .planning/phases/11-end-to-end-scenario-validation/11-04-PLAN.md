    **Release date:** {today's date}

    ### Overview

    Epistract v2.0 transforms the project from a single-domain biomedical knowledge graph tool into a domain-agnostic framework. Plug in a domain schema (YAML + extraction prompts + epistemic rules), point at a document corpus, and get a structured knowledge graph with epistemic analysis.

    - **contracts** — 11 entity types, 11 relation types for event/vendor contract analysis

    ### New Features

    - **Two-Layer Knowledge Graph** — Layer 1 (brute facts) + Layer 2 (epistemic analysis: conflicts, gaps, risks, confidence)
    - **Convention-Based Epistemic Dispatch** — Each domain defines its own epistemic rules; the core pipeline dispatches automatically
    - **Domain-Agnostic Workbench** — Interactive web dashboard works with any domain's graph output
    - **Telegram Bot** — Chat interface for knowledge graph exploration via Telegram Bot API
    - **Standalone Plugin Install** — `claude plugin install epistract` works without repo clone
    - **Regression Suite** — Automated baseline comparison for all scenarios via `make regression`

    - **Three-layer structure:** `core/` (domain-agnostic pipeline), `domains/` (pluggable configurations), `examples/` (consumer applications)
    - **Domain resolver** — `core/domain_resolver.py` discovers and loads domain packages by name or alias
    - **Plugin-first architecture** — All operations via `/epistract:*` slash commands
    - **Consumer decoupling** — Workbench and Telegram bot in `examples/`, work with any domain
