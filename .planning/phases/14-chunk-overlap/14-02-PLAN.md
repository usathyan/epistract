### blingfire.text_to_sentences_and_offsets(text: str) -> tuple[str, list[tuple[int, int]]]
```python
import blingfire

sentences_joined, offsets = blingfire.text_to_sentences_and_offsets(
    "Dr. Smith arrived. He left at noon. The meeting ended."
)
# sentences_joined: "Dr. Smith arrived.\nHe left at noon.\nThe meeting ended."
#                   (sentences separated by '\n')
# offsets: [(0, 18), (19, 35), (36, 55)]  # (start, end) char positions in ORIGINAL text
```

Notes for the helper:
- The second element is a list of `(start_char, end_char)` tuples indexing into the INPUT text.
- Empty input → `("", [])`.
- Single sentence → one tuple.
- The `\n`-joined string in position 0 is informational; we will slice the ORIGINAL text using the offset tuples to preserve the original whitespace (matches D-12 — section_header and surrounding formatting unchanged).

    ```python
    def _sentence_overlap(text: str) -> str:
        """Return the last-N-sentences tail of `text`, capped at OVERLAP_MAX_CHARS.

        Used by all three split paths (_split_at_paragraphs sub-chunks,
        _merge_small_sections ARTICLE-boundary flushes, _split_fixed fallback)
        to bleed context from the end of one chunk into the start of the next,
        so entities and relations whose mentions straddle a chunk boundary
        reach the extractor.

        Behavior (Phase 14 D-02, D-03):
          - Empty `text` → returns "".
          - Fewer than OVERLAP_SENTENCES sentences → returns all sentences.
          - Cumulative length of last-N sentences ≤ OVERLAP_MAX_CHARS → returns
            exactly those N sentences.
          - Cumulative length > OVERLAP_MAX_CHARS → returns the most-recent
            whole sentences that fit under the cap; NEVER mid-sentence
            truncation.
          - A single sentence longer than OVERLAP_MAX_CHARS (pathological
            "whereas…" clause) → returns "" rather than emit a partial
            sentence. Operators auditing provenance see an empty overlap
            and know the cap bit; the extractor still gets the full
            sentence in its OWN chunk.

        The helper is pure: no side effects, no I/O, no module-state
        mutation. Safe to call from any split path.

        Args:
            text: Source text to pull a sentence-level tail from.

        Returns:
            Overlap tail ending on a sentence boundary, or "".
        """
        if not text:
            return ""

        # blingfire returns (sentences_joined_by_\n, [(start, end), ...])
        # where offsets index into the ORIGINAL text. We slice the original
        # text (not the joined string) so whitespace and punctuation that
        # the sentence detector preserved are exact.
        _joined, offsets = blingfire.text_to_sentences_and_offsets(text)
        if not offsets:
            return ""

        # Take the last OVERLAP_SENTENCES sentences (or all, if fewer).
        tail = offsets[-OVERLAP_SENTENCES:]

        # Walk right-to-left, accumulating sentences whose combined length
        # stays under OVERLAP_MAX_CHARS. This guarantees whole-sentence
        # boundaries and favors the MOST RECENT sentences (which carry
        # the boundary-straddling context we care about).
        selected: list[tuple[int, int]] = []
        total = 0
        for start, end in reversed(tail):
            piece = text[start:end]
            piece_len = len(piece)
            # +1 for the joining space we'll insert between sentences
            added = piece_len if not selected else piece_len + 1
            if total + added > OVERLAP_MAX_CHARS:
                break
            selected.append((start, end))
            total += added

        if not selected:
            # Single sentence longer than the cap — refuse mid-sentence cut.
            return ""

        # Re-reverse to document order, then slice original text between
        # (first_start, last_end). This preserves any intra-tail whitespace
        # exactly as it appeared in the source document.
        selected.reverse()
        first_start = selected[0][0]
        last_end = selected[-1][1]
        return text[first_start:last_end]
    ```
