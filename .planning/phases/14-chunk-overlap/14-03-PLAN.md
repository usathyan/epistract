### Chonkie Chunk → epistract chunk dict mapping
For a chonkie `Chunk` at index `i` in `chunker.chunk(buffer_text)`:
- `text` = `chunk.text` (already includes overlap prefix when i > 0)
- `char_offset` = `buffer_offset + chunk.start_index` (absolute doc offset)
- `overlap_prev_chars` = `max(0, prev_chunk.end_index - chunk.start_index)` when i > 0 else 0
- `overlap_next_chars` = `max(0, chunk.end_index - next_chunk.start_index)` when i < len(chunks)-1 else 0
- Section header: `buffer_header if i == 0 else f"{buffer_header} (cont.)"` (D-12 preserved)

### Cross-flush tail cache (M-1/M-2 — CORRECTNESS INVARIANT)
Chonkie's overlap is scoped to a single `chunker.chunk(text)` call. When `_merge_small_sections` hard-flushes at an ARTICLE boundary, the next flush gets a fresh chunker with no memory of the previous article's tail. To carry overlap across flushes, we thread a nonlocal `_pending_tail` cache populated from the RAW buffered body (NOT from the emitted chunk's text, which would chain overlaps across multiple flushes — the M-1/M-2 bug):

```python
def _merge_small_sections(...):
    chunks = []
    buffer_header = buffer_text = ""
    buffer_offset = 0
    _pending_tail = ""  # cross-flush cache

    def _flush():
        nonlocal ..., _pending_tail
        if not buffer_text.strip():
            return
        incoming = _pending_tail  # from cache, NOT chunks[-1]["text"]
        # ... emit chunks using chonkie or small-section path ...
        _pending_tail = _tail_sentences(buffer_text)  # RAW body, not chunks[-1]["text"]
        buffer_header = buffer_text = ""
        buffer_offset = 0
```
