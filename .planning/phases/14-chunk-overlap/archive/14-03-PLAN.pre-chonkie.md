### Cross-flush tail cache (M-1/M-2 fix — CRITICAL CORRECTNESS INVARIANT)
The overlap that a flush emits to the NEXT flush must be computed from the PREVIOUS flush's RAW body text, NOT from `chunks[-1]["text"]`. The emitted `text` already includes an overlap prefix from its own predecessor — re-running `_sentence_overlap` on it would chain overlaps and, in small-article / multi-flush scenarios, bleed content from THREE flushes back instead of ONE.

The fix is a nonlocal `_pending_tail` cache threaded through `_flush`:

```python
def _merge_small_sections(...):
    chunks: list[dict] = []
    buffer_header = ""
    buffer_text = ""
    buffer_offset = 0
    _pending_tail = ""   # cache: tail of previous flush's RAW body

    def _flush() -> None:
        nonlocal buffer_header, buffer_text, buffer_offset, _pending_tail
        if not buffer_text.strip():
            return

        incoming = _pending_tail  # read from cache, not chunks[-1]["text"]

        # ... emit chunks, computing outgoing = _sentence_overlap(body) from RAW body ...

        # Update cache for the NEXT flush:
        _pending_tail = _sentence_overlap(buffer_text)  # RAW body, no overlap prefix

        buffer_header = ""
        buffer_text = ""
        buffer_offset = 0
```

### char_offset calculation contract (D-11, M-6 fix)
For a merged section whose body starts at char 5000 in the original document text, and which we split into 3 sub-chunks of length 4000, 3500, 4200 at paragraph boundaries:
  - sub-chunk 0: char_offset = 5000 (body[0:4000])
  - sub-chunk 1: char_offset = 5000 + (offset of paragraph-2 start within body)
  - sub-chunk 2: char_offset = 5000 + (offset of paragraph-N start within body)

`_split_at_paragraphs` returns `list[{"text": str, "relative_offset": int}]` where `relative_offset` is the char index WITHIN the merged body where this sub-chunk's body starts. `_merge_small_sections` then emits `char_offset = buffer_offset + sub["relative_offset"]`.

**M-6 fix (blank-paragraph drift):** The paragraph-iteration must only advance `current_start` when the FIRST real paragraph joins `current`, not when a blank/whitespace-only paragraph is skipped. See Task 1 action block for the honest pattern.

    ```python
    def _split_at_paragraphs(text: str, max_size: int) -> list[dict]:
        """Split text at paragraph boundaries (double newline) up to max_size.

        Returns a list of {"text": str, "relative_offset": int} dicts where
        `relative_offset` is the char index WITHIN `text` where this sub-chunk
        body starts. Overlap prefixes are NOT added here — callers prepend them.

        The honest per-sub-chunk offset (Phase 14 D-11) lets `_merge_small_sections`
        compute each chunk's absolute char_offset as `buffer_offset + relative_offset`
        instead of all sub-chunks of one oversized section sharing the parent offset.

        Blank/whitespace-only paragraphs are skipped WITHOUT advancing `current_start`
        (Phase 14 M-6): `current_start` is set only when the first real paragraph
        joins the current group, and reset (via `None`) after every flush so the
        next group's start is the next real paragraph, not a preceding blank gap.

        Args:
            text: Source text.
            max_size: Soft upper bound on sub-chunk body length (before overlap).

        Returns:
            List of dicts with keys `text` and `relative_offset`.
            Non-empty on non-empty input; always at least one dict.
        """
        # Fast path: text small enough to stay as one sub-chunk.
        if len(text) <= max_size:
            stripped = text.strip()
            return [{"text": stripped, "relative_offset": 0}] if stripped else []

        # Compute paragraph spans using finditer so we know the exact (start, end)
        # offsets in `text` for each paragraph body — unlike re.split which drops
        # the delimiter and loses positions.
        paragraph_splits = list(re.finditer(r"\n\n+", text))
        boundaries: list[tuple[int, int]] = []
        prev = 0
        for m in paragraph_splits:
            boundaries.append((prev, m.start()))  # paragraph span
            prev = m.end()
        boundaries.append((prev, len(text)))  # last paragraph

        parts: list[dict] = []
        current = ""
        current_start: int | None = None  # None until the first real paragraph joins

        for start, end in boundaries:
            para = text[start:end]
            if not para.strip():
                # Blank/whitespace paragraph: do NOT advance current_start.
                # Skipping here without touching current_start prevents the
                # offset from lagging across blank gaps (M-6).
                continue
            # First real paragraph of this group → anchor current_start.
            if current_start is None:
                current_start = start
                current = para
                continue
            # Would adding this paragraph push us over max_size? If yes, flush.
            if len(current) + len(para) + 2 > max_size:
                parts.append({"text": current.strip(), "relative_offset": current_start})
                current = para
                current_start = start  # new group starts at this real paragraph
            else:
                current = current + "\n\n" + para

        if current.strip() and current_start is not None:
            parts.append({"text": current.strip(), "relative_offset": current_start})

        return parts if parts else [{"text": text.strip(), "relative_offset": 0}]
    ```

    ```python
                "is_overlap_region": False,
            })

        if chunks:

        return chunks
    ```
