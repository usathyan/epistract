## Summary

Seven documentation and configuration files were reviewed for the v2.2.0 documentation refresh. No security vulnerabilities or data-loss risks were found. The plugin manifest files (`plugin.json`, `marketplace.json`) and `CHANGELOG.md` are clean. `CLAUDE.md` is consistent with the codebase.

The main issues are stale content in `DEVELOPER.md` — a large section of the document was not updated when the codebase migrated scripts from `scripts/` to `core/` and command syntax changed from hyphen-separated (`/epistract-setup`) to colon-separated (`/epistract:setup`). ## Warnings

### WR-01: DEVELOPER.md Plugin Component Reference uses obsolete command syntax

**File:** `DEVELOPER.md:451-465`
**Issue:** The "Plugin Component Reference" commands table lists all commands using the old hyphen-separated syntax (`/epistract-setup`, `/epistract-ingest`, `/epistract-validate`, etc.). The current codebase and all other documentation use colon-separated syntax (`/epistract:setup`, `/epistract:ingest`). The two additional bullet references at lines 464 and 637 repeat the same stale syntax. A developer following these instructions would invoke the wrong commands.
**Fix:** Replace all occurrences in the Plugin Component Reference table and surrounding text:
- `/epistract-setup` → `/epistract:setup`
- `/epistract-ingest` → `/epistract:ingest`
- `/epistract-validate` → `/epistract:validate`
- `/epistract-build` → `/epistract:build`
- `/epistract-view` → `/epistract:view`
- `/epistract-query` → `/epistract:query`
- `/epistract-export` → `/epistract:export`
- `/epistract-epistemic` → `/epistract:epistemic`

### WR-04: DEVELOPER.md Skills table and code comment carry stale 23/46 type counts

### WR-05: docs/ADDING-DOMAINS.md contracts example heading has wrong entity/relation counts

**File:** `docs/ADDING-DOMAINS.md:214`
**Issue:** The "Contracts Example" section heading reads "9 entity types, 9 relation types". Every other surface in the codebase (CHANGELOG.md:100, CLAUDE.md:12, CLAUDE.md:70, README.md:384, README.md:444) consistently states the contracts domain has 11 entity types and 11 relation types. The YAML example below the heading actually lists 9 entity types (PARTY, CONTRACT, OBLIGATION, DEADLINE, COST, VENUE, SERVICE, INSURANCE, PENALTY), which does not match the authoritative 11/11 count either — the YAML example is an abbreviated excerpt rather than the full schema. The heading claim of 9/9 is therefore doubly wrong: it contradicts the authoritative count (11/11) and misrepresents even the truncated example.
**Fix:** Update the heading to match all other documentation:
```markdown
#### Contracts Example (11 entity types, 11 relation types)
```
Add a note below the YAML example clarifying it is an excerpt and the full schema in `domains/contracts/domain.yaml` defines 11 types each.

### IN-01: README.md references `make test-integ` but the Makefile target is `test-integration`

**File:** `README.md:427`
**Issue:** The "Testing" subsection of Contributing/Development shows:
```bash

```
